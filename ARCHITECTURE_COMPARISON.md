# Сравнение архитектур: OpenClaw vs OYM Bot

## 🏗️ Общая архитектура

### OpenClaw
```
Единый Gateway процесс (порт 18789)
├── WebSocket RPC (типизированный протокол)
├── HTTP API (OpenAI-совместимый)
├── Control UI (SPA)
└── Статические ассеты

Клиенты:
- TUI (Ink/React)
- Control UI (браузер)
- Mobile apps (iOS/Android)
- Telegram/Discord/WhatsApp/Signal/iMessage
```

### OYM Bot (наша система)
```
Gateway процесс (2 порта: 4000 HTTP + 4001 WS)
├── Socket.io WebSocket
├── REST API
└── Статические ассеты (опционально)

Клиенты:
- CLI/TUI (chalk, простой)
- Web UI (Next.js + визуализация офиса)
- Telegram bot
- (будущее: Discord, WhatsApp)
```

## 🔑 Ключевые различия

| Компонент | OpenClaw | OYM Bot | Преимущество |
|-----------|----------|---------|--------------|
| **Протокол** | WebSocket RPC (кастомный) | Socket.io | OYM: проще, стандартный |
| **TUI** | Ink (React для терминала) | chalk (простой) | OpenClaw: мощнее, OYM: проще |
| **Конфиг** | Hot-reload | Статический markdown | OpenClaw: удобнее |
| **Сессии** | JSONL транскрипты | JSONL + Markdown контекст | OYM: двойное хранение |
| **Фронтенд** | Control UI (дашборд) | Офис-визуализация (игра) | OYM: уникальный UX |
| **Плагины** | npm-пакеты | Встроенные каналы | OpenClaw: расширяемость |
| **Порты** | Один (18789) | Два (4000, 4001) | OpenClaw: проще |

## ✅ Что у нас хорошо

1. **Сессии** - правильная архитектура:
   - `cli:local:agentId` - CLI сессия
   - `telegram:userId:agentId` - Telegram сессия
   - Web выбирает из существующих (не создает свои)

2. **Визуализация** - уникальная фича:
   - Офис с агентами
   - Мини-игра интерфейс
   - Позиционирование агентов

3. **Простота** - легко понять и расширить:
   - Markdown конфиги (AGENT.md, SOUL.md)
   - Простой CLI без React
   - Socket.io вместо кастомного протокола

4. **Команды** - встроенная система:
   - `/model`, `/status`, `/help`
   - Работают во всех каналах
   - Легко добавлять новые

## 📊 Архитектурные решения

### Правильно реализовано ✅

1. **SessionManager** - изолированные сессии по каналам
2. **MemoryManager** - session-aware контекст и транскрипты
3. **AgentRuntime** - единая точка выполнения задач
4. **CommandManager** - универсальные команды для всех каналов
5. **Gateway** - централизованный WebSocket hub

### Соответствует OpenClaw паттернам ✅

1. **Сессии в JSONL** - как в OpenClaw
2. **Агенты в директориях** - как в OpenClaw
3. **Markdown конфиги** - как в OpenClaw (AGENTS.md, SOUL.md)
4. **WebSocket для всех клиентов** - как в OpenClaw
5. **Единый Gateway процесс** - как в OpenClaw
